Phase 10g Bootstrap

Cost dashboard panel

Renders cost-feed.json as a Pipeline Cost section on the live dashboard. Closes out the cost monitoring story.
What it shows:
  • This week: total USD spend, call count, in/out token totals.
  • 12-week trend: inline SVG sparkline of weekly USD totals.
  • 4-week per-workflow breakdown: small table, sorted by spend.
  • Empty-state aware: if cost-feed.json is missing or empty, shows "monitoring active" with a brief explainer instead of broken numbers.

Deploy

Phase 10g lives entirely in the dashboard repo. No paul-job-pipeline changes. The dashboard-push helper does the work: clones fresh, runs the patcher, rebuilds index.html with the new cost section, commits and pushes.

I (Claude) am running the deploy directly from the sandbox via the same path used by the morning-brief cron. You don't need to touch your local dashboard tree for this phase, but the cost section will appear in the next dashboard build.

Done when

The live dashboard at paulwhitaker06.github.io/job-search-dashboard shows a "Pipeline Cost" section near the bottom (between Speculative Outreach and Morning Brief Log). After tomorrow's morning brief and Sunday's first real rollup, the panel populates with real numbers.

What this does NOT touch

  • Cost monitoring logic (logger, rollup workflow). Those are phase 10c, already running.
  • Stat cards in the top row. The cost panel is a separate section, not a new stat card.
  • Mobile layout. The 1fr/2fr grid collapses naturally on narrow screens.

Files

  • patcher.py, the Python script that adds compute_cost_summary and build_cost_section_html functions to build-dashboard.py, plus the call site and the template slot. Idempotent, safe to re-run.
